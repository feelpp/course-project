hifimagnet-1: Statistical Characterization of High-Field Magnet Operations

1. Overview

Level M1 Applied Mathematics (Internship or Semester Project)

Duration

3-4 months (internship) or one semester (project)

Prerequisite knowledge

Multivariate statistics, Python/pandas, basic physics (electromagnetism appreciated but not required)

MagnetDB roadmap mapping

Phase 1 (RecordStats) + Phase 3 (CumulativeStats)

Deliverables

Python module + analysis report

2. Context

LNCMI (Laboratoire National des Champs Magnetiques Intenses) operates resistive high-field magnets capable of producing magnetic fields up to 36 Tesla. These magnets are water-cooled polyhelix inserts composed of concentric helical conductors carrying tens of thousands of amperes.

During operation, a monitoring system records time series of physical quantities: magnetic field, coil currents, voltages, temperatures, pressures, and water flow rates. Each experimental session produces a record - a multivariate time series typically containing 50-70 channels sampled at regular intervals.

MagnetDB is the platform that centralizes this data. It organizes records by site (a physical installation), magnet (an assembly of parts), and part (an individual helix, ring, or current lead). The goal of this project is to define and compute meaningful statistical summaries of operational data at each level of this hierarchy.

3. Scientific Objectives

3.1. 1. Define relevant statistics for operational records

Not all statistical summaries are equally useful for magnet operations. The student must engage with the physical meaning of each measured quantity to determine which statistics are scientifically meaningful.

For example:

  • For the magnetic field: the max and mean are directly relevant (the max determines the experiment’s scientific reach); the ramp-up rate matters for mechanical stress

  • For temperatures: the max outlet temperature indicates thermal load; the temperature differential (Tout - Tin) reflects cooling efficiency; time spent above a threshold may indicate stress

  • For currents: the per-coil current distribution reveals the operating regime; the deviation between coils feeding the same power supply may indicate an electrical issue

  • For pressures and flows: these are health indicators for the cooling system rather than scientific outputs

The student should produce a taxonomy of statistics organized by physical relevance, not just by mathematical definition.

3.2. 2. Compute per-record summaries

For each record, compute a structured summary at three levels:

Site level: aggregate quantities that characterize the full installation during this session - total power, peak field, cooling performance, duration, duty cycle.

Magnet level: quantities specific to one magnet within the site - currents for the helix power supply, field contribution, operating regime identification (ramp-up, plateau, ramp-down, quench).

Part level: quantities specific to an individual helix - its current, voltage, resistance variation (DRcoil), and calibration temperature (Tcal). The resistance variation is particularly important: it tracks conductor aging over the magnet’s lifetime.

3.3. 3. Aggregate across records for lifetime statistics

Given N records spanning the operational lifetime of a site, magnet, or part, compute cumulative statistics:

  • Total operating hours

  • Lifetime max/mean/std of key quantities

  • Which record holds the overall maximum (for traceability)

  • Distribution of operating regimes across records

  • Trend detection: is the resistance variation drifting over time? Is the cooling efficiency degrading?

3.4. 4. Exploratory analysis

With the statistical summaries computed, investigate:

  • Do operational records cluster into distinct regimes? (PCA or simple clustering on summary vectors)

  • Are there correlations between operating conditions and anomalies?

  • Can you identify records that are statistical outliers?

  • For parts that have been in multiple magnet configurations: does the operating history differ?

4. Mathematical Content

  • Descriptive statistics: min, max, mean, std, skewness, kurtosis, quantiles - applied to multivariate time series

  • Correlation analysis: Pearson/Spearman correlation between channels within a record; cross-correlation with time lags for causal analysis (e.g., current increase → temperature increase with delay)

  • Weighted aggregation: when aggregating across records of different durations, define appropriate weighting schemes (time-weighted means vs. record-count means)

  • Non-stationary series: operating records are inherently non-stationary (ramp-up → plateau → ramp-down). The student must decide whether to compute statistics over the full record, over identified phases, or both

  • Phase identification: simple segmentation of the time series into operating phases using change-point detection or threshold-based rules

  • Dimensionality reduction: PCA on the per-record summary vectors to identify dominant modes of operational variation

  • Trend analysis: linear regression on lifetime series of key indicators (resistance drift, max field evolution)

5. Technical Environment

5.1. Provided

  • Docker image with Python 3.11, pandas, numpy, scipy, matplotlib, seaborn, scikit-learn, Jupyter

  • Sample dataset: 15-20 operational records from 2+ magnet configurations, with geometry and material data

  • student_helpers module: record parser, configuration loader

  • Starter notebooks: data loading, basic exploration, multi-record analysis

  • Data README: column reference with units, coil-to-helix mapping, physics context

5.2. Not required

  • Django, MagnetDB, Docker infrastructure knowledge

  • MagnetTools (not needed for raw statistics - only for Project B)

  • Web development, database skills

5.3. Development workflow

The student works in Jupyter notebooks for exploration, then refactors validated code into a Python module with tests. All work is done with pandas DataFrames and Python dicts - no database interaction.

def compute_cumulative_stats(
    record_stats_list: list[dict],
    entity_type: str,   # "site", "magnet", or "part"
    entity_name: str
) -> dict:
    """Aggregate per-record stats into a lifetime summary.

    Args:
        record_stats_list: List of dicts from compute_record_stats(),
            ordered chronologically.
        entity_type: Which level to aggregate.
        entity_name: Name of the entity (e.g., "M9_M19061901" for site).

    Returns:
        Cumulative statistics dict.
    """

7. Suggested Timeline

Period Activities

Weeks 1-2

Onboarding: run starter notebooks, understand the data, read physics background

Weeks 3-4

Design the taxonomy of statistics. Define what to compute at each level. Discuss with supervisor

Weeks 5-7

Implement compute_record_stats(). Test on sample records. Iterate on the output schema

Weeks 8-9

Implement compute_cumulative_stats(). Test on multi-record sequences

Weeks 10-11

Exploratory analysis: clustering, PCA, trend detection

Weeks 12-14

Write report. Clean up code. Final presentation

8. Evaluation Criteria

  • Scientific relevance: Are the chosen statistics physically meaningful? Do they capture what matters for magnet operations?

  • Code quality: Is the module well-structured, tested, documented, and usable as a library?

  • Analysis depth: Does the exploratory analysis reveal genuine patterns in the data?

  • Report quality: Is the mathematical formulation rigorous? Are results clearly presented?
